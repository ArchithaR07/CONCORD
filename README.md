# CONCORD

**Trust-Weighted, Centrality-Aware, Precedence-Resolving, Threat-Mapped, Self-Publishing Policy Intelligence System**

CONCORD is an advanced, multi-stage engine for ingesting, analyzing, resolving, and visualizing organizational policies. It is designed to consume unstructured policy documents, extract their actionable obligations, detect contradictions or redundancies using a cascade of heuristics and LLMs, resolve those conflicts using graph centrality and precedence rules, map the remaining gaps against MITRE ATT&CK, and finally self-publish a unified, "perfect" policy document.

---

##  Architecture (The 16 Layers)

The system runs a sophisticated pipeline (L0 through L16) in a single pass:

- **L0: Ingestion** – Parses `.md` policy files and structural metadata.
- **L1: Extraction** – Extracts granular obligations, scope, and enforcement strength.
- **L2: Lens (Embeddings)** – Computes dense vector embeddings (e.g., `all-MiniLM-L6-v2`) for semantic search.
- **L3: Filter** – Generates high-probability candidate pairs for comparison using topic-bucketing and cosine similarity.
- **L4: Bench (Rules & LLMs)** – Evaluates candidate pairs for `CONFLICT` or `REDUNDANT` relationships using deterministic rules, falling back to an LLM provider cascade (HuggingFace → Groq → Gemini → Mock) for ambiguous cases.
- **L5: Trust Reconcile** – Computes a continuous "Trust Score" combining structural certainty, semantic overlap, and LLM confidence.
- **L6: Arbitrate & Precedence** – Resolves actual conflicts based on policy hierarchy, document authority, and temporal decay.
- **L7: Decay** – Flags policies and obligations that haven't been reviewed recently (Staleness).
- **L8–L12: Graph & Centrality (Mesh, Score, Compass, Voice)** – Builds a directed policy graph, identifies "keystone" load-bearing clauses using PageRank-style centrality, computes organizational policy debt, and generates natural-language explanations.
- **L15: CRUCIBLE (Threat Mapping)** – Maps the operative (surviving) obligations against the MITRE ATT&CK matrix to identify coverage gaps and propose draft clauses.
- **L16: SYNOD (Compilation)** – Assembles a unified, contradiction-free Master Policy Document containing all ratified rules and proposed gap-fillers.

##  Project Structure

The project has been fully unified into two core directories:

```text
CONCORD/
├── a1build/                 # The core Python Backend & Pipeline Engine
│   ├── backend/engine/      # L0-L16 models, rules, and stubs
│   ├── data/policies/       # Drop your Markdown policy files here
│   ├── outputs/             # JSON artifacts generated by the pipeline
│   ├── scripts/             # Helper scripts and legacy audit tools
│   ├── run_pipeline.py      # The master orchestrator script
│   └── requirements.txt     # Python dependencies
│
└── concord-frontend/        # The React/Vite Visual Dashboard
    ├── client/src/          # UI Components, pages, and hooks
    ├── server/              # Express API that serves the a1build outputs
    ├── package.json         # Node dependencies
    └── vite.config.ts       # Vite config + integrated Express Middleware
```

##  Quickstart

### 1. Run the Backend Pipeline

The backend requires Python 3.10+. It reads Markdown files from `a1build/data/policies/` and outputs resolved JSON graphs to `a1build/outputs/`.

```powershell
cd a1build
python -m venv venv
.\venv\Scripts\activate
pip install -r requirements.txt

# Create an .env file for LLM keys (optional, will fallback to mock if omitted)
# GEMINI_API_KEY=...
# GROQ_API_KEY=...
# HF_API_TOKEN=...

# Run the full L0->L16 pipeline
python run_pipeline.py
```

*Note: The pipeline takes ~45 seconds to run on standard mock/cached data, and will generate all JSON artifacts required by the frontend.*

### 2. Run the Frontend Dashboard

The frontend is a React SPA built with Vite and TailwindCSS, served by an integrated Node API that reads from `a1build/outputs/`.

```powershell
cd concord-frontend
npm install

# Start the dashboard on http://localhost:3000
npm run dev
```

### Dashboard Tabs
1. **Ingestion & Extraction (L0-L2):** View raw clauses and semantic embeddings.
2. **Bicameral Analytics (L3-L5):** Scatterplots of Trust Scores and LLM vs Rule verdicts.
3. **Precedence & Mesh (L6-L12):** Interactive Force-Directed Graph of policy relationships, centrality metrics, and automated redundancy resolutions.
4. **Audit & Portals (L15-L16):** 
   - **CRUCIBLE:** Threat coverage map highlighting MITRE ATT&CK gaps.
   - **SYNOD:** The final compiled, living policy document.
